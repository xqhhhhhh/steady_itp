# NOL 抢票插件（Chrome MV3）

## 功能

- 输入演唱会详情页 URL 后自动执行流程：
  1. 点击 `立即购买`
  2. 进入选座页后随机选座
  3. 点击 `完成选择`
  4. 在价格页调整张数并点击 `预购/订购`
  5. 填写手机号（区号 + 手机号）
  6. 勾选同意条款并点击 `总计`
  7. 在 Eximbay 弹窗选择 `WeChat Pay`
  8. 点击确认支付直到出现二维码

- 速度策略：`请求优先 + 点击兜底`
  - 注入 `page-hook.js` 捕获站点请求，尝试快路径重放请求
  - 快路径失败时自动回退到 DOM 点击流程

## 安装

1. 打开 `chrome://extensions/`
2. 开启右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择目录：
   - `/Users/xuqihan/Desktop/code_study/抢票/extension`

## 使用

1. 登录相关账号后，点击插件图标
2. 填写：
   - 演唱会 URL（`https://world.nol.com/...`）
   - 票数
   - 国家区号（默认 `86`）
   - 手机号
3. 点击 `开始`
4. 插件会自动跑到微信二维码页面后停止

## 注意

- 页面文案可能中/繁/英/韩混用，插件已做多语言关键字匹配，但平台改版后仍可能需要调整选择器。
- 普通图片验证码仍按原有 OCR 识别流程处理。
- 滑块验证码现已提供显式开关 `滑块验证码自动拖动`，开启后会自动尝试拖动；若未通过，仍需手动完成后继续自动流程。
- 其他风控弹窗、会员验证等，仍需手动处理后再继续自动流程。

## MCP 调试（Chrome DevTools）

若你在 Codex 中使用 MCP 调试，请在 `~/.codex/config.toml` 中确认已配置：

- `chrome-devtools`
- `chrome-devtools-extension`

配置变更后需要重启 Codex 会话才能生效。

插件内也提供了本地诊断：

- 在 popup 点击 `诊断当前页`
- 若返回 `已连接 content-script`，说明注入成功
- 若诊断失败，通常是当前标签页不在匹配域名或扩展尚未刷新
